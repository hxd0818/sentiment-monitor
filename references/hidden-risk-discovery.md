# 隐匿性风险发现（Hidden Risk Discovery v2.2）

> **核心目标：找到常规搜索找不到的、与已知风险事件存在隐蔽关联的负面信号。**
> **触发时机：发现 L2+ 风险事件后，每轮必须执行（与常规搜索并列的独立步骤）。**

---

## 总原则：隐匿搜索必须"去品牌名"

**所有R策略的Query设计都必须遵守一条铁律：搜索时不含品牌名。**

原因：最危险的隐匿文章往往不直接点名品牌。如果Query中包含品牌名，这些文章永远不会出现在搜索结果中。

| 搜索方式 | 能搜到什么 | 搜不到什么 |
|---------|-----------|-----------|
| `"赛迪顾问" 造假` | 直接点名品牌的文章 | 用"某机构""某评价机构"代称的文章 |
| `"买榜" "百强县"` | 代称文章 + 直接点名文章 | 无 |

**因此，每个R策略的Query都应该有两套：**
- **A套（含品牌名）**：搜直接点名的常规文章
- **B套（去品牌名，用事件指纹）**：**这才是隐匿策略的核心，搜代称/框架转移/关联文章**

**B套才是区分"普通搜索"和"隐匿发现"的关键。如果只做A套不做B套，等于没做隐匿发现。**

---

## 七大策略总览

| # | 策略名 | 一句话说明 | 目标隐蔽层级 |
|---|--------|-----------|-------------|
| **R1** | **语义去伪装** | 找用委婉语/代词/暗示替代直接点名的负面内容 | L1 |
| **R2** | **关联实体辐射** | 通过人/公司/项目/地点间接关联到品牌 | L2 |
| **R3** | **时间窗口异常扫描** | 事件前后时间窗口内的异常聚集信号 | L3 |
| **R4** | **跨域信号融合** | 不同领域(股价/人事/招聘/工商)的同步异常 | L4 |
| **R5** | **渠道下沉探测** | 在非主流渠道(论坛/深帖/评论区)找信号 | L1-L2 |
| **R6** | **叙事框架转移检测** | 检测对事件的叙事角度是否在向更危险的方向转移 | L5 |
| **R7** | **反事实推演** | "如果X发生，Y会怎样"式的潜在风险预判 | L2-L4 |

---

## 各策略操作方法

### R1 语义去伪装

**原理**：负面报道有时不直接点名品牌，而是用"某机构""某咨询公司""业内知名"等代称。
**这是所有隐匿策略的基础——如果R1做不好，R2-R7都会漏。**

**核心原则：R1的Query必须用事件行为指纹搜索，不含品牌名。**

**操作步骤**：
1. 从已知风险事件中提取行为指纹（不是品牌名）：如"买榜""付费排名""数据造假""主观指标调分"
2. 用行为指纹 + 行业/场景关键词组合搜索（**搜索时不含品牌名**）
3. 筛选结果中使用了代称但描述匹配的条目

**R1 必搜Query模板（每个L2+事件都必须执行）**：

| Query组 | 搜索词 | 目标 |
|---------|--------|------|
| **R1-A 行为指纹** | `"{行为动词}" "{行业}"` 如 `"买榜" 咨询` / `"付费排名" 县域` | 直接搜行为 |
| **R1-B 代称+行为** | `"某机构" {行为}` / `"某咨询公司" {行为}` / `"某评价机构" {行为}` | 搜代称指代 |
| **R1-C 委婉语** | `"{委婉词}" {行业}` 如 `"数据优化" 榜单` / `"综合服务" 排名` | 搜话术包装 |
| **R1-D 金额特征** | `"{金额数字}" "{行业}"` 如 "498万 咨询" / "498万 榜单" | 金额是唯一标识 |

**错误做法**：
```
"赛迪顾问" 买榜               -- 代称文章搜不到
"赛迪顾问" 数据造假           -- 同上
```

**正确做法**：
```
"买榜" "百强县"              -- 不含品牌名
"某评价机构" "主观指标"       -- 用代称搜
"498万" "咨询项目"            -- 用金额搜
```

### R2 关联实体辐射

**原理**：通过品牌的关联方（客户/合作伙伴/竞争对手/供应商）间接找到相关负面。

**操作步骤**：
1. 从已有信息中提取关联实体列表（人名/公司名/地名/项目名），这些来自status.json的seeds[]
2. 逐个搜索"实体名 + 负面关键词"
3. 判断该实体的负面是否可辐射到目标品牌

**关键**：status.json 的 seeds[] 中应持续积累关联实体。

### R3 时间窗口异常扫描

**原理**：重大事件前后，相关讨论会出现时间聚集。

**操作步骤**：
1. 确定风险事件的基准日期 T
2. 搜索 [T-7天, T+30天] 窗口内的相关信息
3. 关注：突然出现的评论者 / 异常密集的同主题文章 / 沉默后的首次发声

### R4 跨域信号融合

**原理**：真正的危机往往在多个领域同时出现信号。

**操作步骤（按优先级）**：
1. **股价/财务**：搜索"{品牌} 股价" / 年报变动
2. **人事变动**：搜索"{品牌} 辞职" / 高管变更 / 招聘冻结
3. **工商变更**：搜索"{品牌} 变更" / 法律诉讼
4. **客户/合作方动态**：重要客户是否终止合作 / 发表 distancing 声明
5. **监管/合规**：新的监管动作 / 政策变化

### R5 渠道下沉探测

**原理**：早期负面往往先出现在非主流渠道。

**操作步骤**：
1. 主流媒体之外的平台定向搜索：知乎 / 微博 / 公众号 / 论坛 / 贴吧
2. 深入查看长帖/高赞评论（不只看标题）
3. 关注UGC内容的情感倾向

### R6 叙事框架转移检测（最重要）

**原理**：同一事件被不同框架叙事时，危险程度差异巨大。
- "商业争议"框架 -> 低危险
- "形式主义"框架 -> 中危险
- "新型腐败"框架 -> [高危险]

**核心原则：R6的Query必须使用事件特征词，不能只搜品牌名。**
因为最危险的框架转移文章往往**不直接点名品牌**，而是用"某机构""某咨询公司""某评价机构"等代称，同时把事件包装进更大的叙事体系（如"新型腐败6种类型"）。

**操作步骤**：
1. 从已知风险事件中提取事件指纹（行为+金额+地点+结果，不含品牌名）
2. 用事件指纹 + 框架关键词组合搜索（**搜索时不含品牌名**）
3. 分析搜索结果中的文章用了什么叙事框架
4. 检测框架是否在向更严重的方向转移

**R6 必搜Query模板（每个L2+事件都必须执行以下搜索，全部不加品牌名）**：

| Query组 | 搜索词 | 目标 |
|---------|--------|------|
| **R6-A 腐败框架** | `"{金额}" "{榜单/排名/评价}" 通报` | 搜事件是否被纳入反腐话语 |
| **R6-B 新型腐败** | `"新型腐败" "{行业关键词}"` 或 `"隐形腐败" "{行业关键词}"` | 搜是否被归类为新型腐败案例 |
| **R6-C 典型案例** | `"{事件特征}" 典型案例` 或 `"{事件特征}" 警示教育` | 搜是否进入警示教育/案例库 |
| **R6-D 形式主义** | `"形式主义" "{地名/客户名}" 榜单` 或 `"整治" "{事件特征}"` | 搜是否被作为形式主义典型 |
| **R6-E 代称匹配** | `"某{机构类型}" {事件行为}` 或 `"某评价机构" {事件行为}` | 搜用代称指代品牌的文章 |
| **R6-F 评论框架** | `{事件特征} 本质 是` 或 `{事件特征} 暴露` | 搜深度评论文章的框架定性 |
| **R6-G 新型腐败科普** | `"不收钱不收礼" {行业/事件关键词}` | 搜面向公众的新型腐败科普/警示文（这类文章常把事件作为案例嵌入） |
| **R6-H 分类拆解文** | `"第X种" {事件行为}` 或 `"{数字}种" 腐败 类型` | 搜将事件归类为"第N种XX"的拆解分析文（如"6种腐败类型"） |
| **R6-I 泛社会传播** | `"悄悄蔓延" / "正在体制内" / "值得警惕" {行业关键词}` | 搜泛社会层面的预警式传播（百家号/头条等平台的长文评论） |

> **v2.4新增说明**：R6-G/H/I三个新组别是基于真实漏报案例补充的。关键教训：当事件被中央通报后，自媒体会生产大量"科普式深度分析"，这些文章：(1)不用品牌原名 (2)把事件嵌入了更大的叙事体系 (3)传播量往往超过原始通报。这三组Query专门针对此类文章。
>
> **v2.5新增说明**：R6-G/H/I的搜索结果**必须对高相关URL执行全文抓取**（`--fetch-url`），不可只看snippet就下结论。2026-04-21真实案例：《新型腐败正在体制内悄悄蔓延》一文在snippet中未显示与赛迪顾问的直接关联（因用"某评价机构"代称），但全文第4类"虚假合作"完整引用了海城买榜事件。仅看snippet会漏判此为"不相关"。

**以赛迪顾问百强县买榜事件为例，正确的R6 Query应该是（全部不含"赛迪顾问"）**：
```
"498万" "百强县" "通报"
"新型腐败" "榜单"
"虚假合作" "咨询" "形式主义"
"某评价机构" "主观指标"
"百强县" "典型案例"
"不收钱不收礼" "榜单"
"498万" "咨询项目"
```

**错误做法（会导致漏掉关键文章）**：
```
"赛迪顾问" 新型腐败          -- 文章没提品牌名，搜不到
"赛迪顾问" 典型案例          -- 同上
```

**危险信号清单（出现以下任一即标记为高风险转移）**：
- 党媒/纪委/政法类媒体开始用特定框架引用事件
- 文章标题含"新型腐败""隐形腐败""典型案例""警示教育"等关键词
- 事件被作为某类腐败/违规的"第X种类型"案例进行拆解（如本文的"第4种：虚假合作"）
- 高校/培训机构将事件编入教材/案例库
- 文章从分析商业逻辑转向引用党纪国法条款
- 百家号/今日头条等平台的长文评论将事件纳入更大叙事体系
- **文章使用"不收钱不收礼""悄悄蔓延""值得警惕"等泛社会预警式标题**（v2.4新增）

#### 框架升级检测表（v2.4新增）

**Agent在分析R6搜索结果时，必须用此表判断框架是否在升级：**

| 当前已知最危险框架 | 更危险框架（发现即标记为🔴最高优先级） | 应对动作 |
|-------------------|--------------------------------------|---------|
| 商业争议 | 形式主义典型 | 立即升级风险等级 |
| 形式主义典型 | **新型腐败/隐性腐败案例** | **立即补充二阶追踪，作为独立新条目写入报告** |
| 新型腐败案例 | **具体腐败罪名定性（贪污/受贿/滥用职权）** | **最高警报，可能触发司法程序** |
| 个案讨论 | **纳入全国性教育素材/教材/培训课程** | **长尾风险，持续监控多年** |

> **2026-04-21真实漏报案例**：《新型腐败正在体制内悄悄蔓延》一文将赛迪顾问买榜案列为"6种新型腐败之第4种-虚假合作"的唯一案例。该文：(1)全文无"赛迪顾问" (2)用"某评价机构"代称 (3)叙事框架从"形式主义"升级为"新型腐败典型案例" (4)在百家号等平台广泛传播。因搜索Query未覆盖R6-G/H/I方向，完全漏报。**此案例证明R6-G/H/I三个新组别的必要性。**

### R7 反事实推演

**原理**：基于当前趋势推演"如果...会怎样"。

**操作步骤**：
1. 基于 R1-R6 的发现，识别最危险的演变路径
2. 搜索该路径上已经发生的类似案例（作为"前车之鉴"）
3. 产出格式："如果[条件]继续发展，可能触发[后果]，参考案例：[URL]"

## 隐蔽层级定义

| 层级 | 定义 | 示例 |
|------|------|------|
| **L1** | 语义伪装 -- 用代称/委婉语替代点名 | "某评价机构" |
| **L2** | 实体关联 -- 通过关联方间接涉及 | 同一采购方的其他项目出事 |
| **L3** | 时间共振 -- 时间窗口内异常聚集 | 事件后某话题讨论量突增 |
| **L4** | 跨域映射 -- 多领域同时出现异常 | 股价跌+员工离职+客户解约同时出现 |
| **L5** | 框架转移 -- 叙事角度向危险方向演化 | 从"商业争议"转向"腐败" |

## 产出物

所有发现的隐匿性信号写入 `hidden_risks.json`：

```json
[
  {
    "id": "HR-{N}",
    "signal": "信号摘要",
    "source": "来源平台/媒体",
    "url": "{真实URL或URL缺失}",
    "strategy": "R1-R7",
    "related_event": "RE_ID或风险名",
    "strength": "强/中强/中/弱",
    "concealment_level": "L1-L5",
    "explanation": "为什么相关",
    "discovered_round": N
  }
]
```

报告 6.4 隐匿信源表的数据来源就是此文件。每条记录必须包含url字段。
