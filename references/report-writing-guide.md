# 报告写作指南

> **目标是写出一份高质量的分析报告，不是填表。**
> **8章结构是报告的骨架，内容是你的分析洞察。**

---

## 报告结构（8章）

| # | 章节 | 写什么 |
|---|------|--------|
| 0 | **报告概要** | 一段话概括：监测了什么、发现了什么、最关键的结论是什么 |
| 一 | **核心发现** | 整体舆情评估（正面/中性/负面比例）+ 核心指标 + 最值得关注的1-3个要点 |
| 二 | **详细分析** | 分正面/中性/负面三个方向展开，每条信息带URL和简要分析 |
| 三 | **话题维度分析** | 从产品服务、管理治理、财务业绩、社会责任、争议危机等维度分析 |
| 四 | **风险评估** | 已确认的高风险点 + 潜在风险预警，每项带严重程度和应对思路 |
| 五 | **应对建议** | 短期/中期/长期建议，每条具体可执行（不是"加强管理"这种空话） |
| 六 | **信息源清单** | 正面/中性/负面/隐匿信源四张表，每条带URL |
| 附录 | **附录** | 数据来源说明 + 分析方法 + 免责声明 |

---

## 写作原则

### 内容质量
- **有洞察，不堆砌**：不要把搜索到的所有信息都塞进去。选最有代表性的。
- **有判断，不模棱两可**：作为分析师，给出你的判断和理由。
- **有层次，不扁平**：核心发现放在前面，细节放在后面。

### 格式规范
- 信源只写媒体/网站名称（如"新华网""经济观察网"），**不出现任何搜索工具名称**
- 所有引用的信息必须带URL。找不到就写 `URL缺失: 原因`
- 技术支持固定写：`赛迪网`（放在报告末尾）
- 第六章的四张信息源表与第二章的分析内容对应（提到了就要列出来）

### 隐匿信源（第六章6.4节）
这是本报告区别于普通舆情监测的核心差异化内容。
基于你在分析过程中发现的隐匿信号整理填写：
- 如果发现了框架升级、代称文章、跨域关联等 → 详细列出
- 如果确实没发现 → 写明"经多轮隐匿性扫描未发现额外信号"，并简要说明扫描了哪些方向

---

## 常见陷阱

| 陷阱 | 后果 | 怎么避免 |
|------|------|---------|
| 把所有搜索结果都塞进报告 | 报告冗长无重点 | 只选代表性信息，其余舍弃 |
| 分析全是描述没有判断 | 像搜索引擎不像分析师 | 每节都要有"这意味着什么"的判断 |
| 应对建议写成空话 | 没有实操价值 | 每条建议回答：谁、什么时间前、做什么、做到什么程度 |
| 第六章和正文脱节 | 信息不一致 | 正文提到的信源必须在表中出现 |
| 忽略隐匿信源 | 丢掉核心竞争力 | 6.4节是本skill的差异化价值，认真写 |
