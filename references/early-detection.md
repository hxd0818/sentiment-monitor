# 早期发现 -- 四层漏斗模型（Early Detection v2.1）

> **核心目标：在负面信息爆发前尽早发现，缩短发现延迟。**
> **触发时机：每轮迭代最先执行（Layer 0），发现弱信号后执行 Layer 1-3。**

---

## 四层漏斗

```
Layer 0: 广度嗅探（每轮必做）
    | 发现异常？
Layer 1: 弱信号捕获
    | 通过筛选？
Layer 2: 验证与放大
    | 确认风险？
Layer 3: 风险预警输出
```

---

## Layer 0：广度嗅探（每轮最先执行）

**目的**：用最少的搜索量快速扫描是否有新的高紧急度信号。

### 五行Query（每轮都搜）

| # | Query类型 | 示例(赛迪顾问) | 目的 |
|---|---------|---------------|------|
| 1 | 品牌名精确 | "赛迪顾问" | 最新综合动态 |
| 2 | 产品/服务关键词 | "赛迪 百强县" / "赛迪 榜单" | 核心业务相关 |
| 3 | 高管名 | "赛迪 杨天慈" / "赛迪 赵卫东" | 高管动向/发声 |
| 4 | 行业/客户关键词 | "工信部 赛迪" / "百强县 通报" | 行业监管/政策 |
| 5 | 负面定向 | "赛迪 造假" / "赛迪 通报" / "赛迪 问题" | 直接捕捉负面 |

### 执行规则
- 每轮先用这5个Query快速扫一遍
- 结果存入 `raw/q{round}_L0_*.txt`
- 极速正负分流：看到明显负面 -> 立即标记，不等收敛
- 正常信息 -> 进入常规分析流程

## Layer 1：弱信号捕获

**目的**：识别"看起来不严重但可能变严重"的弱信号。

### 五种弱信号类型

| 类型 | 定义 | 识别方法 |
|------|------|---------|
| **WS-1 语义伪装** | 用委婉语/代词/暗示替代直接点名负面 | "某机构""某咨询公司"+负面语境 |
| **WS-2 渠道先行** | 小平台先出现，大平台尚未跟进 | 论坛/知乎/公众号有负面，主流媒体无 |
| **WS-3 情感漂移** | 同一来源从正面转向中性/负面 | 连续跟踪中发现情感评分下降 |
| **WS-4 框架转移** | 对同一事件的叙事框架变化 | 从"商业争议"->"腐败""形式主义" |
| **WS-5 关联异常** | 关联方出现负面但未直接点名品牌 | 合作方/客户/竞争对手出事 |

### Layer 1 触发条件（满足任一即进入）
- Layer 0 发现任何不确定信号
- 连续2轮情感分布出现明显偏移
- 发现新的关联实体（人/公司/地点）

## Layer 2：验证与放大

**目的**：确认弱信号是否为真实风险。

### 三重验证

| 验证层 | 方法 | 通过标准 |
|--------|------|---------|
| **V1 交叉验证** | 用不同Query/不同源搜同一条信号 | >=2个独立信源确认 |
| **V2 时序验证** | 检查信号的时间趋势 | 不是孤立时间点，有持续或增长 |
| **V3 来源验证** | 评估信源的权威性 | 至少1个B级以上信源支持 |

### 放大操作
- 围绕验证通过的信号扩展搜索（+3~5个衍生Query）
- 存入 `raw/q{round}_L2_{信号简写}.txt`
- 如确认风险 -> 升级到 Layer 3 或触发隐匿性发现

## Layer 3：风险预警输出

**目的**：产出结构化的早期预警。

### 预警格式（写入报告或hidden_risks.json）
```
EARLY_WARNING: {信号摘要}
  - 发现时间: {ISO时间}
  - 弱信号类型: WS-X
  - 可信度: {高/中/低} (基于三重验证结果)
  - 置信度: 0.XX
  - 建议行动: {立即关注/启动深挖/持续监控}
  - 信源: [{URL1}, {URL2}]
```

### 质量目标
| 指标 | 目标值 |
|------|--------|
| 发现延迟 | <=24h（理想<=6h） |
| 弱信号召回率 | >=60% |
| 误报率 | <=30% |

---

## 与其他模块的协作

```
早期发现 Layer 0 -> 每轮最先执行（独立）
早期发现 Layer 1-3 -> 发现弱信号时执行
       | 确认 L2+ 风险
隐匿性发现     -> 并行启动（七大策略横向扫描）
风险深挖       -> 并行启动（单事件纵向钻透）
```
