# 案例研究：赛迪顾问"新型腐败"文章漏报分析

> **从错误中学习。这个案例驱动了 v2.4→v2.5 的关键改进，也是 v3.0 方法论重构的核心动机。**

---

## 事件概述

**时间**：2026-04-21
**任务**：为「赛迪顾问」生成舆情监控报告
**漏报内容**：百家号文章《新型腐败正在体制内悄悄蔓延：不收钱不收礼，但却更隐蔽更加值得警惕》
**该文核心内容**：将赛迪顾问百强县买榜案列为 **"6种新型腐败之第4种——虚假合作型"** 的唯一典型案例

---

## 漏报链条还原

```
1. 中央纪委通报海城买榜事件（2026-01-20）
   ↓
2. 自媒体生产深度分析文章《新型腐败正在体制内悄悄蔓延》
   ↓ 关键特征：
   - 全文从未出现"赛迪顾问"4个字
   - 用"某百强县榜单评价机构"代称
   - 将事件嵌入"6种新型腐败类型"的更大叙事框架
   - 发布平台：百度百家号（泛社会传播渠道）
   ↓
3. 我们的搜索Query全部围绕品牌名（赛迪顾问/CCID/02176）
   ↓
4. 该文章对搜索引擎来说 = 与品牌无关 → 不出现在搜索结果中
   ↓
5. 即使R6搜了"新型腐败 榜单"，返回结果因snippet不显示具体案例而误判为"无新信号"
   ↓
6. 收敛 → 报告中完全缺失此框架升级信息
   ↓
7. 爸爸手动指出该文章 → 触发深度反思和优化
```

## 根因分析（5层）

| # | 根因 | 层级 | 说明 |
|---|------|------|------|
| R1 | **匿名引用盲区** | 数据采集层 | 关键文章用"某...机构"代称，基于品牌名的Query全部失效 |
| R2 | **R6执行表面化** | 策略层 | R6设计初衷就是抓这个，但只搜了"新型腐败 榜单"就停了，没继续展开到变体 |
| R3 | **二阶搜索机制缺失** | 架构层 | 没有"事件变成公共议题后的二次/三次传播追踪"机制 |
| R4 | **Agent分析深度不足** | 判断层 | "同一事件+新框架"被误判为"重复信息"，没识别出框架升级 |
| R5 | **收敛过早** | 流程层 | 3轮就收敛，且R6只做了部分就判定无新信号 |

## 这个案例教给我们什么

### 教训1：中央通报 = 事件性质突变点

当一个品牌相关事件被中央/国家级信源通报后：
- 它不再只是"品牌的危机"
- 它变成了"公共议题"
- 公共议题会被无限次地二次、三次传播
- 这些传播几乎不用品牌原名

**→ 必须有脱品牌名的追踪能力**

### 教训2：snippet ≠ 全文

该文章的snippet（摘要）中没有显示与赛迪顾问的直接关联。
但全文第4类"虚假合作"完整引用了海城买榜事件的所有关键细节。

**→ 对高相关结果必须抓全文再判断**

### 教训3："重复信息" vs "框架升级"

旧思维：同一事件的搜索结果 = 可能是重复信息 → 跳过
新思维：同一事件 + 新叙事框架 = **框架升级信号** → 重点分析

**→ Agent需要具备识别框架升级的分析能力**

### 教训4：硬编码Query模板永远不够

我们在v2.4中加了R6-G/H/I三组新Query模板来"修复"这个问题。
但这只是运气好——这次漏报的关键词恰好能被3组新模板覆盖。

下一次呢？下一个事件可能用完全不同的方式被框架转移。
**没有万能的Query模板清单。只有会思考的Agent。**

**→ 这就是为什么需要v3.0方法论驱动重构**

## 修复验证

在v2.5中加入以下改进后重新运行：

| 改进 | 效果 |
|------|------|
| Step 3E 二阶传播追踪（事件指纹脱品牌名搜索） | ✅ 检测到框架升级信号 |
| R6-G/H/I 三组新Query + 全文抓取要求 | ✅ 命中框架升级检测 |
| 框架升级检测表 + hidden_risks.json独立条目 | ✅ 报告自动包含10处"新型腐败"分析 |

**但v2.5仍然是打补丁式的改进。v3.0从根本上解决了这个问题——通过让Agent自主思考而非执行固定清单。**

## v3.2 修复：脱品牌名追踪升级为必选步骤

**时间**：2026-04-21（第二次漏报后）
**根因确认**：方法论中脱品牌名搜索是可选建议而非强制步骤，Agent在执行时自然回退到带品牌名的安全行为。

**修复内容**：
1. **methodology.md 新增 §七 脱品牌名追踪**（独立章节，非R6子步骤）
2. 定义4个触发条件（中央通报/L2+事件/框架升级/省级纪委）
3. 规定3步执行法：提取事件指纹 → 设计纯指纹Query → 实体归因判断
4. **SKILL.md 迭代循环中加入第⑥步为必选**
5. 错误做法表中新增一条

**与之前修复的本质区别**：
| 之前(v2.5) | 现在(v3.2) |
|-----------|-----------|
| R6的子步骤，可能被跳过 | **独立必选步骤，不可跳过** |
| 加了3组硬编码Query模板 | 不给模板，Agent基于指纹自主设计 |
| 依赖Agent"记得"去做 | 写入迭代循环的标准动作清单 |
| 没有实体归因要求 | 必须对结果做实体归因判断 |
