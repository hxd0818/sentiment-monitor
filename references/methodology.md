# 搜索迭代方法论（Methodology v3.0）

> **核心理念：Agent是分析师，不是操作工。**
> **搜索是思考的延伸，不是思考的替代。**

---

## 一、迭代循环模型

### 1.1 核心循环

```
                    ┌──────────────────────────────┐
                    │                              │
                    ▼                              │
    搜索 → 读结果 → 分析 → 发现 → 思考 → 设计Query ─┘
     ↑                                              │
     └──────────────────────────────────────────────┘
```

**每轮循环只有一件事：基于当前已知信息，决定下一步搜什么。**

这不是一个"完成5个步骤就进入下一轮"的打钩流程。
而是一个**"我看到什么？这意味着什么？我还不知道什么？怎么去知道？"** 的连续思考过程。

### 1.2 循环内的思考链

每次拿到搜索结果后，按以下顺序思考：

```
① 我看到了什么？（事实提取）
   ├── 新实体？（人名、公司、地名、项目名）→ 加入种子池
   ├── 新事件？（发生了什么事）→ 标记为风险信号
   ├── 新信源？（谁在说）→ 记录来源和立场
   └── 已知事件的新进展？（时间线更新）

② 这意味着什么？（意义判断）
   ├── 这个信号的严重程度？
   ├── 它是孤立的还是系统性的一部分？
   ├── 叙事角度是什么？（商业争议？形式主义？腐败？）
   └── 有没有比表面看起来更危险的东西？

③ 我还不知道什么？（盲区识别）
   ├── 这件事还有哪些角度没被报道？
   ├── 谁还没发声但应该发声？（当事人、监管方、竞争对手）
   ├── 用了代称/委婉语的文章可能藏在哪？
   └── 如果这件事变得更严重，会以什么形式出现？

④ 我该怎么去发现？（Query设计）
   ├── 基于③的问题，设计能回答这些问题的搜索词
   ├── 考虑：谁会用什么关键词来讨论这件事？
   ├── 特别注意：脱品牌名的搜索（用事件特征代替品牌名）
   └── 对高相关结果，抓全文读（snippet会骗人）
```

### 1.3 扩散 vs 收敛

**扩散（什么时候继续搜）：**
- 发现了新的实体/事件/角度 → 围绕它展开搜索
- 发现了叙事框架转移的迹象 → 验证是否真的在转移
- 上轮搜索引出了更多问题而非答案 → 继续深挖
- 信息增益仍然为正（新结果带来了之前不知道的信息）

**收敛（什么时候该停）：**
- 连续多轮搜索的结果都是已知信息的重复 → 信息增益趋近零
- 所有已发现的重大风险信号都已完成多维度的交叉验证
- 合理推测：再搜一轮也不会有实质性的新发现
- **收敛是一个判断，不是一个计数器。没有"必须满N轮"或"必须执行完M组Query"这种规则。**

## 二、种子池演化论

### 2.1 种子类型

| 类型 | 说明 | 示例 |
|------|------|------|
| event | 已知事件 | "百强县买榜事件" |
| person | 人物 | "杨天慈"、"海城市某领导" |
| company | 组织 | "海城市政府"、"工信部" |
| keyword | 关键词/概念 | "主观指标调分"、"498万" |
| platform | 信源平台 | "中国经济周刊"、"百家号" |
| fingerprint | 事件指纹（脱品牌名） | "498万+百强县+咨询费+通报" |

### 2.2 种子的生命周期

```
发现 → 激活（围绕它搜索）→ 成熟（已充分探索）→ 衰减（后续轮次降低优先级）
                                    ↓
                              若关联新信号 → 重新激活
```

**原则**：
- 每轮优先选择**信息密度最高**的种子（最有可能产出新信息的方向）
- 不要每轮都从所有种子出发平等搜索——那是机械式的，不是思考式的
- 一个种子连续2轮无新发现 → 降低其优先级，但不完全放弃（可能在其他框架下复活）

### 2.3 种子分裂

当一个大种子被深入挖掘时，它会分裂出更细的子种子：

```
"百强县买榜事件"（大种子）
  ├─ 分裂 → "498万咨询合同"（金额维度）
  ├─ 分裂 → "主观指标调分"（技术维度）
  ├─ 分裂 → "海城市"（地理维度）
  ├─ 分裂 → "多个合作县(市)"（扩展维度——原文暗示的）
  └─ 分裂 → "赛迪顾问内部调查"（应对维度）
```

**Agent应该在分析过程中自然地完成这种分裂，而不是按照预设清单执行。**

## 三、风险发现的思考框架

### 3.1 风险不是二元的是/否

风险发现不是一个"有没有负面信息"的是非题。
而是一个**"这个品牌面临什么样的风险图景？"** 的构图题。

思考维度：
- **显性风险**：直接点名品牌的负面（容易发现）
- **隐匿风险**：不点名品牌但可推断关联的（需要思考）
- **潜在风险**：尚未发生但有迹象的（需要推演）
- **级联风险**：A事件触发B事件的连锁反应（需要联想）

### 3.2 隐匿性发现的思考方式

不要把R1-R7当成7个必须执行的检查项。
把它们当成**7种不同的思考透镜**，用来审视同一个问题："有没有我看不到的负面信号？"

| 透镜 | 核心问题 | 启发式思考 |
|------|---------|-----------|
| **语义伪装 (R1)** | "如果有人想说不点名品牌的负面，他会用什么词？" | 代称、委婉话术、行业黑话、金额代号 |
| **实体辐射 (R2)** | "谁跟这个品牌有关系？这些人/组织有没有出事？" | 客户、供应商、合作伙伴、投资人、监管方 |
| **时间共振 (R3)** | "在这个事件前后，有没有异常的信息聚集？" | 突然密集的同主题文章、沉默后的首次发声 |
| **跨域融合 (R4)** | "除了舆论，其他领域有没有同步异常？" | 股价、人事、工商、客户动态、监管动作 |
| **渠道下沉 (R5)** | "主流媒体之外，有没有人在讨论这事？" | 论坛、公众号、知乎、微博、评论区 |
| **框架转移 (R6)** | "同一件事，别人可能用什么更危险的框架来叙述？" | 商业争议→形式主义→腐败→犯罪→教育素材 |
| **反事实推演 (R7)** | "如果情况恶化，下一步会怎样？有没有先兆？" | 类似案例、行业趋势、监管风向 |

**关键区别**：
- ❌ 旧模式："执行R6策略，搜索这9组Query"
- ✅ 新模式："用R6透镜思考：这个事件的叙事框架可能在往哪个方向转移？→ 我觉得可能向'新型腐败'方向转移 → 设计Query验证这个猜想 → 搜到了 → 分析结果 → 再思考下一个可能的转移方向"

### 3.3 框架升级检测（重要但不是 checklist）

当一个事件被发现后，它的叙事框架可能会随时间升级。

**思考方式**：
1. 当前事件被置于什么叙事框架中？（是谁、用什么语气、在什么平台说的）
2. 这个框架是不是已经比最初发现时更危险了？
3. 有没有迹象表明它正在向更危险的框架迁移？
4. 如果有，我能找到处于新框架下的文章吗？

**常见的框架升级路径（参考，不是穷举）**：
```
行业讨论 → 商业争议 → 行政违规 → 形式主义典型 → 系统性乱象 → 腐败/犯罪案例 → 教育素材/教材
```

**Agent应该根据具体事件自主判断当前所处位置和可能的升级方向，而不是对照固定表格打钩。**

## 四、Query设计哲学

### 4.1 好的Query来自对"谁会在哪里说什么"的理解

设计Query前先问自己：
- **谁**会讨论这个话题？（记者、自媒体、业内人士、受害者、监管者...）
- 他们会**在哪里**说？（什么平台、什么类型的媒体）
- 他们会**用什么词**？（专业术语、口语化表达、情绪化用语、代称）

### 4.2 Query设计的黄金法则

| 法则 | 说明 |
|------|------|
| **脱品牌名** | 最危险的隐匿文章不用品牌名。必须学会用事件指纹搜索 |
| **多表述** | 同一意图用不同表述各搜一次（"买榜"/"付费上榜"/"花钱进排名"） |
| **全文验证** | snippet不可信。高相关结果必须抓全文 |
| **小步快跑** | 每轮3-5个精准Query > 1轮10个大而泛的Query |
| **跟随线索** | 上轮结果中的新线索 = 下轮Query的最佳输入 |

### 4.3 二阶传播追踪（为什么重要 + 怎么想）

**为什么**：中央/国家级通报会让一个"品牌危机"变成"公共议题"。公共议题会在各种框架下被二次、三次传播——而且几乎不再出现品牌原名。

**怎么想**：
1. 这个事件的核心特征是什么？（不含品牌名）
2. 谁会把这类事件作为素材来使用？（自媒体科普文、纪法教育文章、高校案例、培训材料...）
3. 他们会把它放进什么叙事框架里？（新型腐败X种类型、形式主义Y种表现、警示教育Z个案例...）
4. 这些人会用什么标题/关键词来写这类文章？

**然后基于以上思考，自主设计Query。不是从模板里选。**

## 五、数据质量与信源判断

### 5.1 不是所有搜索结果等价

| 维度 | 高价值信号 | 低价值信号 |
|------|-----------|-----------|
| 来源 | 央媒/纪委/权威财经 | 转载号/营销号/内容农场 |
| 深度 | 原创深度调查/评论 | 通稿式转发/标题党 |
| 框架 | 引入新叙事角度 | 重复已知信息 |
| 时间 | 首发/早期传播 | 滞后转载 |
| 关联性 | 直接涉及或强间接关联 | 仅提及品牌名但无关负面 |

### 5.2 全文抓取决策

不需要抓所有URL的全文。抓取标准：
- **应该抓**：snippet显示与事件指纹高度相关 / 来自高价值信源 / 标题暗示有深度分析
- **可以不抓**：明显是通稿转发 / 与任何已知事件无关 / snippet就能确认无关
