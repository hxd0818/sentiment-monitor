# 搜索迭代方法论（Methodology v2.1）

> **精简版：搜索->分析->发现->更新的循环。**

---

## 搜索循环

```
Round N:
  (1) 读 status.json（获取当前状态）
  (2) 设计本轮Query（基于种子池 + 早期发现Layer0五行 + 上轮新发现）
  (3) 执行搜索 -> 结果存入 raw/q_{N}_*.txt
  (4) 分析结果：
     |- 提取新实体 -> 加入种子池seeds[]
     |- 识别情感信号 -> 更新情感分布
     |- 发现 L2+ 风险事件？
        |- 是 -> 触发隐匿性发现(R1-R7) + 风险深挖(五步法) + 早期发现(L1-L3)
        └- 否 -> 继续
  (5) 写回 status.json（round, seeds, signals, timestamps）
  (6) 判断是否收敛：
     |- round >= 6 且 连续2轮无新信号 -> 收敛，进入写报告阶段
     └- 否 -> Round N+1
```

## Query设计规则

| 规则 | 正确做法 | 错误做法 |
|------|---------|---------|
| 单问题单搜 | 每个Query只问一件事 | 一个Query包含多个不相关的问题 |
| 关键词数量 | 每个Query 2-5个关键词 | 超过10个关键词的长Query |
| 时间锚定 | 重要Query加时间范围，如"2026年4月" | 不限定时间范围导致结果过时 |
| 语义变体 | 同一意图用不同表述各搜一次 | 只搜一种表述就下结论 |
| 负面定向 | 每轮至少有1-2个负面定向Query | 只搜中性/正面关键词 |

## 种子池管理

**种子类型**：event(事件) / person(人物) / company(公司) / keyword(关键词) / platform(平台)

**更新时机**：每轮分析后，将新发现的实体追加到 status.json 的 seeds[] 数组末尾。

**种子使用**：下一轮的Query设计基于当前种子池中的实体扩展新搜索词。

## 收敛判断（何时停止搜索开始写报告）

| 条件 | 说明 |
|------|------|
| 最低轮次 | round >= 6 才允许收敛（不足6轮必须继续） |
| 信号枯竭 | 连续2轮无新的 L1+ 信号 |
| 风险覆盖 | 已知L2+事件均已完成深挖+隐匿扫描 |

三个条件全部满足才可收敛。任一不满足则继续下一轮。

## 时效分级（数据质量标注）

在报告中标注数据时效时使用：

| 标记 | 含义 | 在报告中的处理方式 |
|------|------|-------------------|
| [FRESH] | 信息发布时间 <= 6个月前 | 可作为核心依据，不需特别标注 |
| [CURRENT] | 信息发布时间 6-18个月前 | 可以使用，但在信息后标注"[CURRENT]" |
| [OLD] | 信息发布时间 18-36个月前 | 仅作背景参考，标注"[OLD]" |
| [OBSOLETE] | 信息发布时间 > 36个月前 | 基本不用，除非是唯一可用信源则标注"[OBSOLETE]并说明原因]" |
